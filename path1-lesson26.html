Lesson 26: Logging in Python

1. What is Logging?

Logging is the process of recording messages that describe what a program is doing during execution.

Logging helps in debugging and monitoring production applications.

2. Why Not Use print()?

  • No log levels
  • Cannot easily disable
  • No timestamps
  • Not suitable for production

3. logging Module

Python provides a built-in module called logging.

4. Logging Levels

Level Purpose
DEBUG Detailed diagnostic information
INFO General information
WARNING Something unexpected
ERROR Serious problem
CRITICAL Program may stop

5. Basic Logging Example

import logging

logging.basicConfig(level=logging.INFO)

logging.info("Program started")
logging.warning("Low disk space")
logging.error("Something went wrong")

6. Logging to a File

import logging

logging.basicConfig(
  filename="app.log",
  level=logging.DEBUG
)

logging.debug("Debug message")
logging.info("Info message")
Logs will be stored in app.log.

7. Log Message Format

logging.basicConfig(
  format="%(asctime)s - %(levelname)s - %(message)s"
)

8. Logging Inside Functions

def divide(a, b):
  if b == 0:
    logging.error("Division by zero")
    return None
  return a / b

9. logging vs print()

print() logging
Simple output Structured messages
No levels Multiple levels
Not configurable Highly configurable

10. Best Practices

  • Use proper log levels
  • Do not log sensitive data
  • Use file logging in production
  • Avoid excessive logging
Never use print() for debugging in production code.

Practice Tasks
