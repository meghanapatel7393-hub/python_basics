Lesson 27: Virtual Environments & pip

1. What is pip?

pip is Python’s package manager used to install, update, and manage external libraries.

Example: Django, Flask, NumPy, Pandas are installed using pip.

2. Why Do We Need Virtual Environments?

  • Avoid dependency conflicts
  • Different projects need different package versions
  • Keep system Python clean
  • Industry standard practice
Never install project libraries globally.

3. What is a Virtual Environment?

A virtual environment is an isolated Python environment with its own Python interpreter and packages.

4. Creating a Virtual Environment

python -m venv venv

This creates a folder named venv.

5. Activating Virtual Environment

Windows

venv\Scripts\activate

Mac / Linux

source venv/bin/activate
Activated environment name appears in terminal.

6. Deactivating Virtual Environment

deactivate

7. Installing Packages with pip

pip install requests

8. Viewing Installed Packages

pip list

9. requirements.txt

requirements.txt stores project dependencies.

pip freeze > requirements.txt
pip install -r requirements.txt

10. Virtualenv vs venv

venv virtualenv
Built-in Third-party
Python 3.3+ Supports older Python
Recommended Advanced features

11. Common Mistakes

  • Forgetting to activate venv
  • Installing packages globally
  • Not sharing requirements.txt
