Lesson 5: Exception Handling & Custom Exceptions

1. What is an Exception?

An exception is a runtime error that disrupts normal program execution.

print(10 / 0)

Output:

ZeroDivisionError
Interview line:
“Exceptions are runtime errors handled using try-except blocks.”

2. Why Exception Handling?

  • Prevents program crash
  • Improves reliability
  • Graceful error handling

3. try & except Block

try:
  x = int(input("Enter number: "))
  print(10 / x)
except ZeroDivisionError:
  print("Cannot divide by zero")
except ValueError:
  print("Invalid input")

4. else Block

else runs only if no exception occurs.

try:
  print("No error")
except:
  print("Error occurred")
else:
  print("Successfully executed")

5. finally Block

finally always executes, error or no error.

try:
  file = open("test.txt")
except FileNotFoundError:
  print("File not found")
finally:
  print("Cleanup done")
Used for resource cleanup.

6. Raising Exceptions

age = -5
if age < 0:
  raise ValueError("Age cannot be negative")

7. Custom Exceptions

Create your own exception by inheriting from Exception.

class InvalidAgeError(Exception):
  pass

def check_age(age):
  if age < 18:
    raise InvalidAgeError("Underage user")

check_age(15)
Custom exceptions improve code clarity.

8. Common Python Exceptions

Exception Cause
ZeroDivisionError Divide by zero
TypeError Wrong type operation
ValueError Invalid value
IndexError Invalid index
KeyError Missing dictionary key
Avoid using bare except:. Always catch specific exceptions.
