1. Functions ‚Äì What Happens Internally?

A function is an object in Python. It has its own memory, scope, and reference.

def greet():
  print("Hello")

print(type(greet))

Output:

<class 'function'>

2. First-Class Functions (Interview Favorite)

  • Functions can be assigned to variables
  • Functions can be passed as arguments
  • Functions can be returned from other functions
def add(a, b):
  return a + b

x = add
print(x(2, 3))

3. Scope in Python

Scope decides where a variable can be accessed.

Types of Scope

  • Local
  • Enclosing
  • Global
  • Built-in
This is called the LEGB Rule.

4. LEGB Rule Explained (VERY IMPORTANT)

x = "global"

def outer():
  x = "enclosing"
  def inner():
    x = "local"
    print(x)
  inner()

outer()

Output:

local
Python searches variables in this order:
Local ‚Üí Enclosing ‚Üí Global ‚Üí Built-in

5. Global Keyword

x = 10

def change():
  global x
  x = 20

change()
print(x)

Output:

20
Avoid excessive use of global in real projects.

6. Closures (INTERVIEW GOLD üî•)

A closure is a function that remembers variables from its enclosing scope even after that scope is finished.

def outer(msg):
  def inner():
    print(msg)
  return inner

fn = outer("Hello Python")
fn()

Output:

Hello Python
Interview line:
‚ÄúClosures allow a function to remember the environment in which it was created.‚Äù

7. Why Closures Are Used?

  • Data hiding
  • Function factories
  • Decorator foundation
