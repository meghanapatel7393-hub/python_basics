Lesson 24: Asynchronous Programming (async / await)

1. What is Asynchronous Programming?

Asynchronous programming allows a program to perform other tasks while waiting for long-running operations like network calls or file I/O.

Async programming is best for I/O-bound tasks.

2. Synchronous vs Asynchronous

Synchronous Asynchronous
Blocking Non-blocking
Waits for task to finish Runs other tasks while waiting
Slower for I/O Faster for I/O

3. async and await Keywords

  • async defines a coroutine function
  • await pauses execution until task completes
import asyncio

async def say_hello():
  print("Hello")

asyncio.run(say_hello())

4. Example: Async with Delay

import asyncio

async def task():
  print("Start")
  await asyncio.sleep(2)
  print("End")

asyncio.run(task())
await asyncio.sleep() is non-blocking.

5. Running Multiple Async Tasks

import asyncio

async def task(name):
  print(f"Start {name}")
  await asyncio.sleep(2)
  print(f"End {name}")

async def main():
  await asyncio.gather(task("A"), task("B"))

asyncio.run(main())
Tasks run concurrently, not sequentially.

6. Async vs Threading vs Multiprocessing

Async Threading Multiprocessing
Single thread Multiple threads Multiple processes
I/O-bound I/O-bound CPU-bound
No GIL issues GIL affected No GIL

7. Real-World Use Cases

  • Web APIs (FastAPI, aiohttp)
  • Web scraping
  • Chat applications
  • High-performance servers

8. Common Mistakes

  • Using blocking code inside async functions
  • Forgetting await
  • Calling async function without event loop
Never use blocking functions like time.sleep() inside async code.
