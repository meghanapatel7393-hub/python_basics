Lesson 31: Python Security Basics & Best Practices

1. Why Security Matters?

  • Protect user data
  • Prevent hacking & data leaks
  • Maintain trust
  • Critical for production systems
Security is not optional — it is mandatory in real-world applications.

2. Common Security Risks

Risk Description
Hardcoded secrets Passwords/API keys in code
Injection attacks SQL or command injection
Insecure inputs No input validation
Weak passwords Plain text storage

3. Never Hardcode Secrets

Hardcoding passwords is a serious security issue.
# ❌ Wrong
DB_PASSWORD = "mypassword"

# ✅ Correct
import os
DB_PASSWORD = os.getenv("DB_PASSWORD")

4. Use Environment Variables

export DB_PASSWORD="secret123"
Environment variables keep secrets outside source code.

5. Password Hashing

Never store passwords in plain text. Always hash them.

import hashlib

password = "mypassword"
hashed = hashlib.sha256(password.encode()).hexdigest()
print(hashed)

6. Input Validation

age = input("Enter age: ")
if not age.isdigit():
  print("Invalid input")
Validate all user inputs to prevent attacks.

7. Avoid eval()

eval() can execute malicious code.
# ❌ Dangerous
eval("2 + 2")

# ✅ Safe
int("2") + int("2")

8. Secure File Handling

with open("data.txt", "r") as file:
  content = file.read()
Always close files properly to avoid leaks.

9. Use HTTPS & Secure Libraries

  • Always use HTTPS
  • Use trusted libraries
  • Update dependencies regularly

10. Security Best Practices

  • Follow least privilege principle
  • Log security events
  • Handle errors safely
  • Review code for vulnerabilities
