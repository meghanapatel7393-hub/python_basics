Lesson 19: Logging & Debugging

1. Why Logging?

Logging helps record events and debug information instead of using print(). It’s better for production-ready applications.

Logging can include timestamp, severity level, and message.

2. Basic Logging Setup

import logging
logging.basicConfig(level=logging.INFO)
logging.info("This is an info message")
logging.warning("This is a warning")
logging.error("This is an error")

3. Logging Levels

Level Description
DEBUG Detailed information, mostly for developers
INFO Confirmation that things are working as expected
WARNING Indication of a potential problem
ERROR Serious problem that prevented some function
CRITICAL Severe error, program may not continue

4. Logging to a File

logging.basicConfig(filename="app.log", level=logging.INFO,
format='%(asctime)s:%(levelname)s:%(message)s')
logging.info("Application started")
logging.error("An error occurred")
Logs can now be saved to a file with timestamps and levels.

5. Debugging with pdb

Python debugger (pdb) allows step-by-step execution to find bugs.

import pdb

def add(a, b):
  return a + b

pdb.set_trace() # Start debugger
result = add(5, 7)
print(result)
Use commands: n (next), c (continue), l (list), p (print variable)

6. Best Practices

  • Use logging instead of print in production
  • Use proper logging levels
  • Store logs in files for auditing/debugging
  • Use pdb or IDE debugger for step-by-step debugging

Practice Tasks

  1. Set up logging to console with INFO and WARNING messages
  2. Set up logging to a file with timestamps
  3. Debug a function using pdb to check variable values
  4. Explain difference between logging levels in an interview
