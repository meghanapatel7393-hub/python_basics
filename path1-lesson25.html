Lesson 25: Python Testing (unittest & pytest)

1. What is Testing?

Testing is the process of checking whether your code works correctly and gives expected output.

Companies expect developers to write testable and reliable code.

2. Why Testing is Important?

  • Find bugs early
  • Improve code quality
  • Prevent future issues
  • Confidence while refactoring

3. Types of Testing

  • Unit Testing – Test small functions
  • Integration Testing – Test modules together
  • System Testing – Test full application

4. unittest (Built-in Python Module)

unittest is Python’s built-in testing framework, inspired by Java’s JUnit.

5. Simple unittest Example

import unittest

def add(a, b):
  return a + b

class TestMath(unittest.TestCase):
  def test_add(self):
    self.assertEqual(add(2, 3), 5)

if __name__ == "__main__":
  unittest.main()
Each test method must start with test_.

6. Common unittest Assertions

Assertion Purpose
assertEqual Check equality
assertTrue Check condition is true
assertFalse Check condition is false
assertRaises Check exception

7. pytest (Popular Third-Party Tool)

pytest is a modern testing framework that is simpler and more powerful than unittest.

Most companies prefer pytest.

8. Simple pytest Example

def add(a, b):
  return a + b

def test_add():
  assert add(2, 3) == 5
No class needed. Just use assert.

9. Running pytest

pytest

10. unittest vs pytest

unittest pytest
Built-in Third-party
More boilerplate Simple syntax
Class-based Function-based

11. Best Practices

  • Write tests early
  • Test edge cases
  • Keep tests simple
  • Use meaningful test names
Never deploy code without tests in real projects.
