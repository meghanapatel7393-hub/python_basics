Lesson 10: Modules, Packages & Virtual Environments

1. What is a Module?

A module is a Python file containing functions, variables, or classes.

# math_utils.py
def add(a, b):
  return a + b
import math_utils
print(math_utils.add(2, 3))
Interview line:
“A module is a single Python file used to organize code.”

2. Types of Modules

  • Built-in modules (math, sys, os)
  • User-defined modules
  • Third-party modules

3. Import Statements

import math
print(math.sqrt(16))
from math import sqrt
print(sqrt(25))
import math as m
print(m.pi)
Avoid from module import * in production.

4. What is a Package?

A package is a folder containing multiple modules.

mypackage/
│── __init__.py
│── file1.py
│── file2.py
Packages help structure large applications.

5. __init__.py

__init__.py marks a directory as a Python package.

It can also execute package-level initialization code.

6. Virtual Environment (venv)

A virtual environment is an isolated Python environment for project-specific dependencies.

python -m venv venv
# Activate (Windows)
venv\Scripts\activate
# Activate (Mac/Linux)
source venv/bin/activate
Prevents dependency conflicts.

7. pip & Requirements

pip install requests
pip freeze > requirements.txt
pip install -r requirements.txt
Used for project dependency management.

8. Interview Comparison

Concept Description
Module Single Python file
Package Collection of modules
venv Isolated environment
