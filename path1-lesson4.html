Lesson 4: Comprehensions & Generators

1. What is Comprehension?

Comprehension is a compact way to create collections using a single line of code.

Interview line:
“Comprehensions provide a concise and readable way to create collections.”

2. List Comprehension

Traditional way:

numbers = [1, 2, 3, 4]
squares = []
for n in numbers:
  squares.append(n * n)

Using list comprehension:

squares = [n * n for n in numbers]
Faster and more Pythonic.

3. List Comprehension with Condition

even_numbers = [n for n in range(10) if n % 2 == 0]

Output:

[0, 2, 4, 6, 8]

4. Dictionary Comprehension

numbers = [1, 2, 3, 4]
square_dict = {n: n * n for n in numbers}

Output:

{1: 1, 2: 4, 3: 9, 4: 16}
Dictionary keys must be immutable.

5. Set Comprehension

nums = [1, 2, 2, 3, 4, 4]
unique = {n for n in nums}

Output:

{1, 2, 3, 4}

6. Generator Expressions

Generator expressions create values on demand instead of storing all values in memory.

gen = (n * n for n in range(5))
print(next(gen))
print(next(gen))
Generators are memory efficient.

7. List vs Generator (Interview Comparison)

Feature List Generator
Memory High Low
Speed Faster access Slightly slower
Reusable Yes No
Once a generator is exhausted, it cannot be reused.
