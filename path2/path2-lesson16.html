Lesson 16: Flask Email Sending & Notifications

1. Why Email Notifications?

Sending emails is essential for user verification, password reset, and notifications in web apps.

2. Setting Up Flask-Mail

pip install Flask-Mail
Flask-Mail is a simple extension for sending emails from Flask applications.

3. Configuring Flask-Mail

from flask import Flask
from flask_mail import Mail, Message

app = Flask(__name__)
app.config["MAIL_SERVER"] = "smtp.gmail.com"
app.config["MAIL_PORT"] = 587
app.config["MAIL_USE_TLS"] = True
app.config["MAIL_USERNAME"] = "your_email@gmail.com"
app.config["MAIL_PASSWORD"] = "your_app_password"
mail = Mail(app)
For Gmail, generate an App Password for security instead of using your main password.

4. Sending a Simple Email

@app.route("/send-mail")
def send_mail():
  msg = Message("Hello from Flask",
    sender="your_email@gmail.com",
    recipients=["recipient_email@gmail.com"])
  msg.body = "This is a test email from Flask."
  mail.send(msg)
  return "Email sent successfully!"

5. Notes & Best Practices

  • Never hardcode passwords; use environment variables.
  • Use background tasks (Celery or Flask-RQ) for sending bulk emails.
  • Ensure proper SMTP configuration for your email provider.
