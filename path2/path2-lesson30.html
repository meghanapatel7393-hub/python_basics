Lesson 30: Flask REST API Deployment to Production

1. Why Deployment?

Deployment makes your Flask API accessible to users on the internet or within a network. Production deployment requires proper server, security, and performance considerations.

2. Using Gunicorn with Flask

# Install Gunicorn
pip install gunicorn

# Run Flask app with Gunicorn
gunicorn -w 4 -b 0.0.0.0:8000 app:app
-w 4 sets 4 worker processes. 0.0.0.0:8000 binds to all network interfaces.

3. Using Nginx as Reverse Proxy

# Sample Nginx configuration
server {
  listen 80;
  server_name example.com;

  location / {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
}
Nginx handles static files, SSL termination, and load balancing.

4. Notes & Best Practices

  • Use environment variables for secrets and configuration
  • Enable HTTPS for secure communication (Let's Encrypt or paid SSL)
  • Monitor app using logs, uptime checks, and metrics
  • Use process managers like Supervisor or systemd to manage Gunicorn
  • Consider scaling with Docker and Kubernetes for larger applications
