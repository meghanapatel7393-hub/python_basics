Lesson 12: Flask Error Handling & Custom Error Pages

1. Why Error Handling?

Handling errors gracefully ensures users see friendly messages instead of raw server errors.

Common errors: 404 (Page Not Found), 500 (Server Error)

2. Basic Error Handling in Flask

from flask import Flask, render_template
app = Flask(__name__)

@app.errorhandler(404)
def page_not_found(e):
  return render_template("404.html"), 404

@app.errorhandler(500)
def server_error(e):
  return render_template("500.html"), 500
Flask allows custom functions for different HTTP error codes.

3. Creating Custom Error Pages

<!-- templates/404.html -->
<h1>404 - Page Not Found</h1>
<p>Sorry, the page you are looking for does not exist.</p>

<!-- templates/500.html -->
<h1>500 - Server Error</h1>
<p>Something went wrong on the server.</p>

4. Testing Error Pages

  • Visit a non-existent route → triggers 404 page
  • Create an intentional error in a route → triggers 500 page

5. Logging Errors (Optional)

import logging
logging.basicConfig(filename="error.log", level=logging.ERROR)

@app.errorhandler(500)
def server_error(e):
  logging.error(f"Server Error: {e}")
  return render_template("500.html"), 500
Logs errors to a file for debugging later.

Practice Tasks

  1. Create 404 and 500 error pages
  2. Trigger errors manually to test pages
  3. Add a route that raises an exception to test 500
  4. Optionally, log errors to a file
