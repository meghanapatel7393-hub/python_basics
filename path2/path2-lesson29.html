Lesson 29: Flask REST API Testing with Pytest

1. Why Testing?

Testing ensures your API works correctly and prevents bugs when updating code. Pytest is a popular Python testing framework.

2. Installing Pytest

pip install pytest pytest-flask

3. Setting Up a Test File

# test_app.py
import pytest
from app import app

@pytest.fixture
def client():
  with app.test_client() as client:
    yield client

4. Writing a Test Case

def test_get_todos(client):
  response = client.get("/todos")
  assert response.status_code == 200
  data = response.get_json()
  assert isinstance(data, list)
Use client.get(), client.post(), etc. to simulate API requests.

5. Running Tests

pytest test_app.py
Pytest will automatically discover test functions prefixed with test_.

6. Notes & Best Practices

  • Test both success and failure scenarios
  • Use fixtures for reusable setup (like client, database)
  • Keep tests small and focused
  • Run tests frequently during development

Practice Tasks

  1. Create tests for all CRUD endpoints
  2. Test error scenarios (invalid input, 404, etc.)
  3. Run tests and ensure all pass
  4. Optional: Integrate tests into CI/CD pipeline
