Lesson 27: Flask REST API Caching with Flask-Caching

1. Why Caching?

Caching improves API performance by storing frequently accessed data in memory, reducing database queries and response time.

2. Installing Flask-Caching

pip install Flask-Caching

3. Configuring Flask-Caching

from flask import Flask, jsonify
from flask_caching import Cache

app = Flask(__name__)
cache = Cache(app, config={'CACHE_TYPE': 'SimpleCache', 'CACHE_DEFAULT_TIMEOUT': 60})
SimpleCache stores data in memory; for production, use Redis or Memcached.

4. Caching an Endpoint

@app.route("/todos")
@cache.cached(timeout=120) # Cache this endpoint for 2 minutes
def get_todos():
  todos = [{"id":1,"task":"task1"},{"id":2,"task":"task2"}]
  return jsonify(todos)
First request fetches from database; subsequent requests within 2 minutes are served from cache.

5. Notes & Best Practices

  • Cache only read-heavy endpoints
  • Invalidate cache when data changes
  • Use distributed caching (Redis) in production for scalability
  • Set appropriate cache timeout based on data freshness requirements
