Lesson 21: Flask REST API Pagination & Filtering

1. Why Pagination & Filtering?

APIs may return large datasets. Pagination limits the number of records per request and filtering allows clients to query specific data.

2. Pagination Example

@app.route("/todos", methods=["GET"])
def get_todos():
  page = int(request.args.get("page", 1))
  per_page = int(request.args.get("per_page", 5))
  todos_query = Todo.query.paginate(page=page, per_page=per_page)
  result = [{"id": t.id, "task": t.task} for t in todos_query.items]
  return jsonify({
    "page": page,
    "per_page": per_page,
    "total": todos_query.total,
    "todos": result
  })
Clients can use query params like ?page=2&per_page=10 to navigate.

3. Filtering Example

@app.route("/todos/filter", methods=["GET"])
def filter_todos():
  search = request.args.get("task", "")
  todos_filtered = Todo.query.filter(Todo.task.contains(search)).all()
  result = [{"id": t.id, "task": t.task} for t in todos_filtered]
  return jsonify(result)
Filtering uses SQLAlchemy query operators. Clients can pass ?task=Flask to search.

4. Notes & Best Practices

  • Always validate page and per_page parameters.
  • Use indexes in database for faster filtering on large datasets.
  • Combine pagination with filtering for efficient API responses.
