Lesson 11: Flask Blueprints & Modular App Structure

1. What are Blueprints?

Blueprints in Flask allow you to organize your app into smaller, modular components for better maintainability.

Useful for large projects with multiple routes and functionalities.

2. Project Structure Example

project_folder/
│── app.py
│── /auth
│  __init__.py
│  routes.py
│── /blog
│  __init__.py
│  routes.py
│── /templates
│── /static
Each module (auth, blog) is a Blueprint.

3. Creating a Blueprint

# auth/routes.py
from flask import Blueprint, render_template
auth_bp = Blueprint("auth", __name__, url_prefix="/auth")

@auth_bp.route("/login")
def login():
  return render_template("login.html")

4. Register Blueprint in main app

# app.py
from flask import Flask
from auth.routes import auth_bp

app = Flask(__name__)
app.register_blueprint(auth_bp)

if __name__ == "__main__":
  app.run(debug=True)
The auth routes are now accessible via /auth/login

5. Advantages of Blueprints

  • Separation of concerns: routes, templates, and static files grouped logically
  • Reusability: Blueprints can be reused in multiple projects
  • Better maintainability for large apps
  • Supports REST APIs and traditional web routes together
