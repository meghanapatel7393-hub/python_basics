Phase 2 – Flask Basics

Lesson 9: Flask CRUD Operations

1. What is CRUD?

CRUD stands for Create, Read, Update, Delete. These are the basic operations in any database-driven application.

2. Simple In-Memory Example

from flask import Flask, jsonify, request
app = Flask(__name__)

todos = [
  {"id": 1, "task": "Learn Flask"},
  {"id": 2, "task": "Build API"}
]

# Read all todos
@app.route("/todos")
def get_todos():
  return jsonify(todos)

3. Create (POST)

@app.route("/todos", methods=["POST"])
def add_todo():
  data = request.get_json()
  new_todo = {"id": len(todos)+1, "task": data["task"]}
  todos.append(new_todo)
  return jsonify(new_todo)
Use POST request to add a new todo.

4. Update (PUT)

@app.route("/todos/<int:id>", methods=["PUT"])
def update_todo(id):
  data = request.get_json()
  for todo in todos:
    if todo["id"] == id:
      todo["task"] = data["task"]
      return jsonify(todo)
  return jsonify({"error": "Todo not found"}), 404

5. Delete (DELETE)

@app.route("/todos/<int:id>", methods=["DELETE"])
def delete_todo(id):
  for todo in todos:
    if todo["id"] == id:
      todos.remove(todo)
      return jsonify({"message": "Deleted"})
  return jsonify({"error": "Todo not found"}), 404

6. Notes

  • In-memory data is temporary; restarting server clears it.
  • For real apps, connect Flask with a database like SQLite, MySQL, or MongoDB.
  • CRUD operations map to HTTP methods: POST → Create, GET → Read, PUT → Update, DELETE → Delete
