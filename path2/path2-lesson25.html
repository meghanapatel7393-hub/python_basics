Lesson 25: Flask REST API Rate Limiting & Throttling

1. Why Rate Limiting & Throttling?

Rate limiting prevents abuse of your API by restricting the number of requests a client can make in a given time period. Throttling protects server resources and improves security.

2. Installing Flask-Limiter

pip install Flask-Limiter

3. Configuring Flask-Limiter

from flask import Flask, jsonify
from flask_limiter import Limiter
from flask_limiter.util import get_remote_address

app = Flask(__name__)
limiter = Limiter(app, key_func=get_remote_address)
get_remote_address identifies clients by their IP address.

4. Applying Rate Limits to Endpoints

@app.route("/todos")
@limiter.limit("5 per minute") # Limit to 5 requests per minute per client
def get_todos():
  return jsonify({"message": "This is a rate-limited endpoint"})
You can change the limit to any number and duration, e.g., "100 per hour".

5. Notes & Best Practices

  • Set reasonable limits for public and authenticated endpoints
  • Combine with authentication to enforce per-user limits
  • Consider using Redis for distributed rate-limiting in production
  • Return meaningful error messages when limit is exceeded

Practice Tasks

  1. Apply rate limiting to all GET and POST endpoints
  2. Test exceeding the limit using Postman
  3. Optional: Use Redis for distributed limiting
  4. Optional: Apply different limits for free vs premium users
