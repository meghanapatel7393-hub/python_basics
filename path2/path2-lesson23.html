Lesson 23: Flask REST API Search Functionality

1. Why Search Functionality?

Search functionality allows clients to find specific data in large datasets quickly using keywords or phrases.

2. Basic Search Implementation

@app.route("/todos/search", methods=["GET"])
def search_todos():
  query = request.args.get("q", "")
  todos = Todo.query.filter(Todo.task.contains(query)).all()
  result = [{"id": t.id, "task": t.task} for t in todos]
  return jsonify(result)
Use query parameter ?q=Flask to search tasks containing the word "Flask".

3. Combining Search with Pagination

@app.route("/todos/search_paginate", methods=["GET"])
def search_paginate():
  query = request.args.get("q", "")
  page = int(request.args.get("page", 1))
  per_page = int(request.args.get("per_page", 5))
  todos_paginated = Todo.query.filter(Todo.task.contains(query)).paginate(page=page, per_page=per_page)
  result = [{"id": t.id, "task": t.task} for t in todos_paginated.items]
  return jsonify({
    "page": page,
    "per_page": per_page,
    "total": todos_paginated.total,
    "todos": result
  })
Combining search with pagination improves performance and usability on large datasets.

4. Notes & Best Practices

  • Use SQL indexes on searchable columns for faster queries
  • Sanitize input to prevent SQL injection
  • Consider adding full-text search or using ElasticSearch for complex search requirements
