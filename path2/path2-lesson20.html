Lesson 20: Flask REST API with Role-Based Access Control

1. What is Role-Based Access Control (RBAC)?

RBAC allows restricting access to certain API endpoints based on user roles such as admin, editor, or user.

2. Adding Role to User Model

class User(db.Model):
  id = db.Column(db.Integer, primary_key=True)
  username = db.Column(db.String(80), unique=True, nullable=False)
  password = db.Column(db.String(200), nullable=False)
  role = db.Column(db.String(20), nullable=False, default="user")

3. Decorator for Role Checking

from functools import wraps
from flask_jwt_extended import get_jwt_identity, jwt_required

def role_required(required_role):
  def decorator(func):
    @wraps(func)
    @jwt_required()
    def wrapper(*args, **kwargs):
      user_id = get_jwt_identity()
      user = User.query.get(user_id)
      if user.role != required_role:
        return {"error": "Access forbidden"}, 403
      return func(*args, **kwargs)
    return wrapper
  return decorator

4. Example Protected Endpoint

@app.route("/admin-only")
@role_required("admin")
def admin_only():
  return {"message": "Welcome Admin!"}
Only users with role admin can access this route.

5. Notes & Best Practices

  • Assign roles at user registration or via admin panel
  • Use JWT identity to fetch user role
  • Combine RBAC with endpoint-specific checks for better security
