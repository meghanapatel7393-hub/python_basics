Lesson 28: Flask REST API Logging & Monitoring

1. Why Logging & Monitoring?

Logging helps track API requests, errors, and behavior. Monitoring allows tracking API health and performance to prevent downtime and detect issues early.

2. Basic Logging Setup in Flask

import logging
from flask import Flask

app = Flask(__name__)
logging.basicConfig(level=logging.INFO) # Log INFO and above
logger = logging.getLogger(__name__)

3. Logging API Requests & Responses

from flask import request

@app.before_request
def log_request():
  logger.info(f"Request: {request.method} {request.url}")

@app.after_request
def log_response(response):
  logger.info(f"Response: {response.status}")
  return response
Logs show every request and response for debugging and monitoring.

4. Monitoring Tools

  • Flask Monitoring Dashboard (FMD)
  • Prometheus + Grafana for metrics
  • NewRelic or Datadog for performance monitoring

5. Notes & Best Practices

  • Log at appropriate levels: DEBUG, INFO, WARNING, ERROR
  • Mask sensitive information in logs
  • Use structured logging for easier parsing
  • Set up alerting on errors and high latency

Practice Tasks

  1. Add logging to all API endpoints
  2. Test by making requests and checking logs
  3. Optional: Integrate Flask Monitoring Dashboard
  4. Optional: Setup alerts for error responses
