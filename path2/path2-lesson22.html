Lesson 22: Flask REST API Sorting & Ordering

1. Why Sorting & Ordering?

Sorting allows clients to receive API data in a specific order, e.g., ascending, descending, by creation date, or by name.

2. Sorting Example with SQLAlchemy

@app.route("/todos/sort", methods=["GET"])
def sort_todos():
  order_by = request.args.get("order_by", "id")
  direction = request.args.get("direction", "asc")
  if direction == "desc":
    todos_sorted = Todo.query.order_by(getattr(Todo, order_by).desc()).all()
  else:
    todos_sorted = Todo.query.order_by(getattr(Todo, order_by).asc()).all()
  result = [{"id": t.id, "task": t.task} for t in todos_sorted]
  return jsonify(result)
Clients can pass ?order_by=task&direction=desc to get descending order by task.

3. Notes & Best Practices

  • Validate order_by field to prevent SQL injection
  • Combine with pagination and filtering for large datasets
  • Provide default sorting if parameters are missing
