Lesson 2: What is Backend & REST API

1. Backend Overview

Backend is the server-side part of a web application that:

  • Processes requests from clients
  • Performs business logic
  • Handles database operations
  • Returns responses to clients

2. Frontend vs Backend

Frontend Backend
HTML, CSS, JS Python, Node.js, Java
User interface Server logic & database
Runs in browser Runs on server

3. What is REST API?

REST (Representational State Transfer) API is a way for a client to communicate with a server using HTTP methods.

REST API allows frontend and backend to be separate.

4. HTTP Methods in REST

  • GET: Read data
  • POST: Create data
  • PUT/PATCH: Update data
  • DELETE: Delete data

5. REST API Example

GET /users → Get list of users
POST /users → Create a new user
PUT /users/1 → Update user with ID 1
DELETE /users/1 → Delete user with ID 1

6. Why REST API?

  • Separation of frontend and backend
  • Supports multiple clients (web, mobile)
  • Stateless & scalable
  • Industry standard
