Lesson 23: Multithreading & Multiprocessing

1. Why Concurrency?

Concurrency helps programs perform multiple tasks efficiently by overlapping execution.

  • Improve performance
  • Handle multiple tasks simultaneously
  • Better resource utilization

2. Multithreading

Multithreading allows multiple threads to run within the same process. Best suited for I/O-bound tasks.

import threading

def task():
  print("Task running")

t = threading.Thread(target=task)
t.start()
t.join()
Threads share memory and resources.

3. Global Interpreter Lock (GIL)

GIL allows only one thread to execute Python bytecode at a time.

Multithreading does NOT improve performance for CPU-bound tasks in Python.

4. Multiprocessing

Multiprocessing runs tasks in separate processes, bypassing the GIL. Best for CPU-bound tasks.

from multiprocessing import Process

def task():
  print("Process running")

p = Process(target=task)
p.start()
p.join()

5. Thread vs Process

Thread Process
Shared memory Separate memory
Lightweight Heavyweight
I/O-bound tasks CPU-bound tasks
Affected by GIL Not affected by GIL

6. Thread Synchronization (Lock)

import threading

lock = threading.Lock()

def task():
  with lock:
    print("Critical section")

t1 = threading.Thread(target=task)
t2 = threading.Thread(target=task)
t1.start()
t2.start()
Locks prevent race conditions.
