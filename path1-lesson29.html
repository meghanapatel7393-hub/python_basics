Lesson 29: Error Handling & Custom Exceptions

1. What is an Error?

An error is a problem that occurs during program execution and stops normal flow.

Errors are also called exceptions in Python.

2. Common Python Errors

Error Reason
ZeroDivisionError Dividing by zero
TypeError Wrong data type
ValueError Invalid value
IndexError Invalid index
KeyError Missing dictionary key

3. Why Handle Errors?

  • Prevent program crash
  • Show user-friendly messages
  • Improve application reliability
  • Important for production systems

4. try / except

try:
  x = 10 / 0
except ZeroDivisionError:
  print("Cannot divide by zero")

5. Handling Multiple Exceptions

try:
  a = int("abc")
except ValueError:
  print("Invalid number")
except TypeError:
  print("Wrong type")

6. else Block

Executes when no exception occurs.

try:
  x = 10 / 2
except ZeroDivisionError:
  print("Error")
else:
  print("Success")

7. finally Block

Always executes (used for cleanup).

try:
  file = open("data.txt")
except FileNotFoundError:
  print("File not found")
finally:
  print("Closing resources")

8. Raising Exceptions

age = -5
if age < 0:
  raise ValueError("Age cannot be negative")

9. Custom Exceptions

You can create your own exceptions for business logic.

class InvalidAgeError(Exception):
  pass

def check_age(age):
  if age < 18:
    raise InvalidAgeError("Not eligible")
  return "Eligible"
Custom exceptions improve code clarity.

10. Best Practices

  • Catch specific exceptions
  • Avoid empty except blocks
  • Use logging instead of print
  • Use custom exceptions for business rules
Never hide errors silently.
