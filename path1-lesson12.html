Lesson 12: Decorators

1. What is a Decorator?

A decorator is a function that takes another function and extends its behavior without explicitly modifying it.

Interview line:
“Decorators help in code reusability and applying cross-cutting concerns like logging.”

2. Simple Decorator Example

def decorator(func):
  def wrapper():
    print("Before function")
    func()
    print("After function")
  return wrapper

@decorator
def say_hello():
  print("Hello World")

say_hello()
@decorator is syntactic sugar for say_hello = decorator(say_hello)

3. Decorators with Arguments

def decorator(func):
  def wrapper(name):
    print("Before function")
    func(name)
    print("After function")
  return wrapper

@decorator
def greet(name):
  print(f"Hello {name}")

greet("Bhavesh")

4. Chaining Decorators

def deco1(func):
  def wrapper():
    print("Deco1")
    func()
    return
  return wrapper

def deco2(func):
  def wrapper():
    print("Deco2")
    func()
    return
  return wrapper

@deco1
@deco2
def hello():
  print("Hello")

hello()
Executes inside-out: deco2 → deco1 → hello

5. Built-in Decorators

  • @staticmethod – Function inside class without self
  • @classmethod – Receives class instead of object
  • @property – Access method like attribute

6. Why Use Decorators?

  • Logging
  • Authentication & Authorization
  • Timing functions
  • Code reusability & clean code

7. Interview Comparison

Concept Use
Decorator Extend behavior of function/class
@staticmethod Class method without self
@classmethod Method receives class instead of instance
@property Method acts like attribute
