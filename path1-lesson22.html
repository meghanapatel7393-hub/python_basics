Lesson 22: Context Managers & with Statement

1. What is a Context Manager?

A context manager is an object that manages resources like files, database connections, or locks by automatically handling setup and cleanup.

Context managers ensure that resources are properly released even if an error occurs.

2. Problem Without Context Manager

file = open("data.txt", "r")
content = file.read()
file.close()
If an error occurs before close(), the file remains open.

3. Using Context Manager (with)

with open("data.txt", "r") as file:
  content = file.read()
# file is automatically closed
Python automatically calls cleanup code when the block exits.

4. How Context Managers Work Internally

  • __enter__() → Setup resource
  • __exit__() → Cleanup resource

5. Creating Custom Context Manager (Class)

class MyContext:
  def __enter__(self):
    print("Enter context")
    return self

  def __exit__(self, exc_type, exc_value, traceback):
    print("Exit context")

with MyContext():
  print("Inside block")

6. Context Manager Using contextlib

from contextlib import contextmanager

@contextmanager
def my_context():
  print("Enter")
  yield
  print("Exit")

with my_context():
  print("Inside")
This approach is cleaner and interview-friendly.
