Lesson 28: Python Packaging & Project Structure

1. What is a Python Package?

A Python package is a directory that contains multiple Python modules and a special file named __init__.py.

Packages help organize large projects into manageable parts.

2. Module vs Package

Module Package
Single .py file Folder with modules
Small functionality Large application

3. Basic Package Structure

myproject/
│── app/
│  │── __init__.py
│  │── main.py
│  │── utils.py
│── tests/
│  │── test_main.py
│── requirements.txt
│── README.md

4. Role of __init__.py

__init__.py tells Python that a directory is a package.

from .utils import add

5. Importing from Packages

from app.utils import add
from app import main

6. Entry Point: main.py

if __name__ == "__main__":
  print("Application Started")
Prevents code from running on import.

7. tests Folder

All test files should be placed inside a tests directory.

pytest automatically detects tests inside this folder.

8. requirements.txt

Lists all project dependencies.

9. setup.py (Introduction)

from setuptools import setup, find_packages

setup(
  name="myproject",
  version="1.0",
  packages=find_packages()
)
Used for packaging and distribution.

10. Good Project Practices

  • Keep code modular
  • Separate logic and tests
  • Use meaningful folder names
  • Write README.md
Avoid writing everything in one file.
