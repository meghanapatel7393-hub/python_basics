Lesson 15: Multithreading & Multiprocessing

1. Why Multithreading?

Multithreading allows Python to run multiple threads (tasks) concurrently, useful for I/O bound tasks.

Threads share memory space but may need synchronization for shared data.

2. Multithreading Example

import threading

def print_numbers():
  for i in range(5):
    print(i)

t1 = threading.Thread(target=print_numbers)
t2 = threading.Thread(target=print_numbers)

t1.start()
t2.start()

t1.join()
t2.join()
start() begins the thread, join() waits for completion.

3. Thread Synchronization

import threading
counter = 0
lock = threading.Lock()

def increment():
  global counter
  for _ in range(1000):
    with lock:
      counter += 1
Locks prevent race conditions in shared resources.

4. Multiprocessing

Multiprocessing runs multiple processes in parallel, bypassing Python’s GIL, useful for CPU-bound tasks.

from multiprocessing import Process

def print_numbers():
  for i in range(5):
    print(i)

p1 = Process(target=print_numbers)
p2 = Process(target=print_numbers)

p1.start()
p2.start()

p1.join()
p2.join()
Each process has its own memory space.

5. Thread vs Process

Aspect Thread Process
Memory Shared Separate
Overhead Low High
Use I/O bound tasks CPU bound tasks
GIL Impact Yes No

6. Queue for Thread Communication

import threading, queue
q = queue.Queue()

def producer():
  for i in range(5):
    q.put(i)

def consumer():
  while not q.empty():
    item = q.get()
    print(item)

t1 = threading.Thread(target=producer)
t2 = threading.Thread(target=consumer)

t1.start()
t1.join()
t2.start()
t2.join()
Queue helps safely share data between threads.
